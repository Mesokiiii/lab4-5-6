## 1. Введение

Анализ сетевых структур является классической задачей теории графов. В данном проекте трубопроводная сеть формализуется в виде неориентированного графа $G = (V, E)$, где множество вершин $V$ представляет собой насосные станции и узлы распределения, а множество ребер $E$ — соединяющие их трубопроводы. Целью разработки является практическое применение алгоритмов поиска в ширину и глубину, алгоритма Дейкстры для поиска кратчайших путей, алгоритма Прима для построения минимального остовного дерева, а также методов поиска точек сочленения.

## 2. Теоретическая база и описание алгоритмов

### 2.1. Лабораторная работа №4: Построение графа и алгоритмы обхода

В данной работе сеть рассматривается как невзвешенный неориентированный граф. Реализованы следующие алгоритмы:

*   **Поиск в ширину (Breadth-First Search, BFS).** Алгоритм обхода графа, использующий структуру данных «очередь» (Queue). Позволяет найти кратчайший путь (по количеству ребер) от стартовой вершины до всех достижимых.
    Вычислительная сложность: $\mathcal{O}(|V| + |E|)$.
*   **Поиск в глубину (Depth-First Search, DFS).** Алгоритм обхода графа, реализованный посредством рекурсии (неявно использующий стек вызовов). Применяется для анализа структуры графа.
    Вычислительная сложность: $\mathcal{O}(|V| + |E|)$.
*   **Поиск компонент связности.** Компонента связности — это максимальный связный подграф. Поиск осуществляется путем итеративного применения алгоритма BFS для каждой непосещенной вершины графа.

### 2.2. Лабораторная работа №5: Взвешенный граф и алгоритм Дейкстры

Граф расширяется до взвешенного: $G = (V, E, W)$, где $W: E \to \mathbb{Z}^+$ — весовая функция, определяющая «длину» или «сопротивление» трубопровода.

*   **Алгоритм Дейкстры.** Применяется для нахождения кратчайших путей от выделенной вершины (источника) до всех остальных вершин графа. 
    Математическая модель релаксации ребра $(u, v)$:
    $$d(v) = \min \{ d(v), d(u) + w(u, v) \}$$
    где $d(x)$ — текущая оценка кратчайшего расстояния до вершины $x$. В реализации используется линейный поиск минимума в множестве непосещенных вершин.

### 2.3. Лабораторная работа №6: Дополнительный анализ графа

В данной работе реализованы алгоритмы, направленные на анализ пропускной способности и отказоустойчивости сети:

*   **Поиск точек сочленения (Articulation Points).** Точка сочленения — это вершина, удаление которой увеличивает количество компонент связности графа. Для поиска используется модифицированный алгоритм DFS, вычисляющий массивы времени обнаружения ($disc$) и минимального достижимого времени ($low$).
    Условие точки сочленения (для не-корня дерева обхода):
    $$low[v] \ge disc[u]$$
    где $u$ — анализируемая вершина, $v$ — ее потомок.
*   **Минимальное остовное дерево (МОД, алгоритм Прима).** Остовное дерево $T \subseteq E$ связного взвешенного графа, минимизирующее функционал:
    $$W(T) = \sum_{e \in T} w(e) \to \min$$
    Алгоритм Прима реализован с использованием приоритетной очереди (`SortedSet` в C#).
*   **Оптимальный путь транспортировки (Задача о максимальном узком месте).** Модификация алгоритма Дейкстры для поиска пути с максимальной пропускной способностью. Релаксация обновляется по правилу максимизации минимума на пути:
    $$C(v) = \max \{ C(v), \min(C(u), w(u, v)) \}$$
    где $C(x)$ — максимальная возможная пропускная способность пути от источника до $x$.

## 6. Руководство пользователя

После запуска приложения `MyApp.WPF` пользователю доступно главное окно, разделенное на логические вкладки:

1.  **Вкладка «Лабораторная №4»:** Требуется нажать кнопку «Загрузить граф». После успешной загрузки из файла `pipeline_network.txt`, становятся активными функции обхода графа (BFS, DFS) и вычисления компонент связности. Результаты выводятся в текстовое поле.
2.  **Вкладка «Лабораторная №5»:** Требуется нажать кнопку «Загрузить взвешенный граф» (файл `pipeline_weighted.txt`). Пользователю предоставляются выпадающие списки (ComboBox) для выбора стартового и конечного узла с целью выполнения алгоритма Дейкстры.
3.  **Вкладка «Лабораторная №6»:** Использует данные, загруженные на предыдущем этапе. Позволяет инициировать поиск точек сочленения, построение МОД (алгоритм Прима) и расчет пути с максимальной пропускной способностью.
4.  **Вкладка «Эксперимент»:** Предназначена для эмпирической оценки временной сложности алгоритмов. Загружает файл `pipeline_large.txt` и производит замеры времени выполнения методами класса `System.Diagnostics.Stopwatch`. Сравнивает время обработки базового и масштабированного графов.

## 7. Формат входных данных

Программа считывает топологию сети из текстовых файлов в кодировке UTF-8. Каждая строка описывает одно ребро. Пустые строки игнорируются.

**Формат невзвешенного графа (Лабораторная работа №4):**
Синтаксис: `<Узел 1> - <Узел 2>`
*Пример:*
```text
НС_Московская - Тверской_трубопровод
Тверской_трубопровод - Калининский_узел
```

**Формат взвешенного графа (Лабораторные работы №5 и №6):**
Синтаксис: `<Узел 1> - <Узел 2>, <Вес>`
*Пример:*
```text
НС_Московская - Тверской_трубопровод, 100
Тверской_трубопровод - Калининский_узел, 90
```

## 8. Тестирование программного обеспечения

Для обеспечения отказоустойчивости разработан набор модульных тестов с использованием фреймворка xUnit. 

Запуск тестов осуществляется командой:
```bash
dotnet test
```

Архитектура тестов разделена на две категории:
1.  **Позитивное тестирование (`Lab4Tests.cs`, `Lab5Tests.cs`, `Lab6Tests.cs`):** Проверка корректности работы алгоритмов на валидных данных (корректный расчет МОД, точное восстановление путей алгоритмом Дейкстры, проверка ацикличности графов).
2.  **Негативное тестирование / Обработка граничных случаев (`Lab4NegativeTests.cs`, `Lab5NegativeTests.cs`, `Lab6NegativeTests.cs`):** Проверка устойчивости системы к аномальным входным данным:
    *   Попытка поиска путей к несуществующим узлам.
    *   Обработка файлов с нарушенным синтаксисом или нулевым весом.
    *   Изолированные вершины и полностью несвязные графы.
    *   Экстремально большие веса ребер (проверка на отсутствие арифметического переполнения).
